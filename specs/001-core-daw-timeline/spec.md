# Feature Specification: Core DAW Timeline

**Feature Branch**: `001-core-daw-timeline`
**Created**: 2026-03-31
**Status**: Draft
**Input**: User description: "Phase 1 코어 DAW - 멀티트랙 타임라인, 기본 재생/정지, 트랙 추가/삭제, 리전 배치, 줌/스크롤, 기본 믹서(볼륨/팬)"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 타임라인에서 트랙 생성 및 재생 (Priority: P1)

사용자가 앱을 열면 빈 타임라인이 보인다. "트랙 추가" 버튼을 눌러 MIDI 트랙을 생성하고, 재생 버튼을 누르면 재생 헤드가 타임라인을 따라 이동한다. 정지 버튼을 누르면 재생이 멈추고 재생 헤드가 현재 위치에 정지한다. 처음 위치로 돌아가려면 정지를 한 번 더 누르거나 Home 키를 사용한다.

**Why this priority**: 타임라인과 트랙은 DAW의 최소 기능 단위이다. 이것이 없으면 다른 모든 기능이 의미를 가질 수 없다.

**Independent Test**: 앱을 열고 트랙을 추가한 뒤 재생/정지를 눌러 재생 헤드가 움직이고 멈추는 것을 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 앱이 로드된 초기 상태, **When** "트랙 추가" 버튼을 클릭, **Then** 타임라인에 새 트랙 행이 나타나고 트랙 헤더에 이름/볼륨/팬 컨트롤이 표시된다
2. **Given** 트랙이 1개 이상 존재하는 상태, **When** 재생 버튼을 클릭, **Then** 재생 헤드가 왼쪽에서 오른쪽으로 BPM에 맞춰 이동하고 위치 표시기(마디:박자)가 실시간 갱신된다
3. **Given** 재생 중인 상태, **When** 정지 버튼을 클릭, **Then** 재생 헤드가 현재 위치에서 멈추고 재생 상태가 해제된다
4. **Given** 정지된 상태, **When** 정지 버튼을 다시 클릭, **Then** 재생 헤드가 1:1(맨 처음)으로 이동한다

---

### User Story 2 - 리전 생성 및 배치 (Priority: P2)

사용자가 타임라인의 빈 영역을 더블클릭하면 해당 위치에 빈 리전(블록)이 생성된다. 리전을 드래그하면 시간 축(가로)과 트랙 간(세로) 이동이 가능하다. 리전의 좌우 끝을 드래그하면 길이를 조절할 수 있다. 리전을 선택 후 Delete 키를 누르면 삭제된다.

**Why this priority**: 리전은 오디오/MIDI 데이터를 담는 컨테이너로, 음악을 구성하는 기본 편집 단위이다. 트랙 위에서 리전을 배치할 수 없으면 어레인지가 불가능하다.

**Independent Test**: 타임라인에서 더블클릭으로 리전을 만들고, 드래그로 위치를 이동시킨 뒤, Delete 키로 삭제할 수 있다.

**Acceptance Scenarios**:

1. **Given** 트랙이 존재하는 타임라인, **When** 빈 영역을 더블클릭, **Then** 해당 위치에 기본 길이(4마디)의 빈 리전이 생성된다
2. **Given** 리전이 존재하는 상태, **When** 리전을 드래그, **Then** 마디 단위 스냅에 맞춰 시간 축으로 이동하거나 다른 트랙으로 이동할 수 있다
3. **Given** 리전이 선택된 상태, **When** 리전의 우측 끝을 드래그, **Then** 리전의 길이가 마디 단위로 늘어나거나 줄어든다
4. **Given** 리전이 선택된 상태, **When** Delete 키를 누름, **Then** 리전이 타임라인에서 제거된다

---

### User Story 3 - 줌 및 스크롤 (Priority: P3)

사용자가 마우스 휠을 스크롤하면 타임라인이 가로(시간 축)로 스크롤된다. Cmd(또는 Ctrl) + 마우스 휠로 줌 인/아웃하여 타임라인의 시간 해상도를 변경한다. 줌 레벨에 따라 그리드 눈금(마디/박자/세분)이 적절히 표시된다. 세로 스크롤로 트랙 목록을 탐색한다.

**Why this priority**: 줌과 스크롤은 타임라인을 탐색하는 기본 조작으로, 곡이 길어지거나 트랙이 많아질 때 필수적이다.

**Independent Test**: 마우스 휠로 타임라인을 좌우로 스크롤하고, Cmd+휠로 줌 인/아웃하여 그리드 눈금이 변하는 것을 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 타임라인이 표시된 상태, **When** 마우스 휠을 위/아래로 스크롤, **Then** 타임라인이 시간 축(가로)으로 스크롤된다
2. **Given** 타임라인이 표시된 상태, **When** Cmd + 마우스 휠로 줌 인, **Then** 시간 해상도가 높아져 더 세밀한 눈금(16분음표 등)이 표시된다
3. **Given** 트랙이 화면보다 많은 상태, **When** 트랙 영역에서 세로 스크롤, **Then** 트랙 목록이 위/아래로 탐색된다
4. **Given** 줌 인된 상태에서 재생 중, **When** 재생 헤드가 화면 끝에 도달, **Then** 타임라인이 자동으로 스크롤되어 재생 헤드를 따라간다

---

### User Story 4 - 기본 믹서 (볼륨/팬) (Priority: P4)

각 트랙 헤더에 볼륨 슬라이더와 팬 노브가 표시된다. 볼륨을 조절하면 해당 트랙의 출력 크기가 변하고, 팬을 조절하면 좌/우 스테레오 위치가 변한다. 뮤트 버튼으로 트랙을 무음 처리하고, 솔로 버튼으로 해당 트랙만 들을 수 있다.

**Why this priority**: 믹서는 여러 트랙의 밸런스를 잡는 기본 기능으로, 멀티트랙의 의미를 완성한다. 다만 P1~P3이 먼저 동작해야 믹서가 의미를 가진다.

**Independent Test**: 여러 트랙에 리전을 배치하고 재생하면서, 각 트랙의 볼륨/팬/뮤트/솔로를 조작하여 소리의 변화를 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 트랙이 존재하는 상태, **When** 볼륨 슬라이더를 왼쪽으로 드래그, **Then** 해당 트랙의 출력 볼륨이 감소하고 슬라이더 값(dB)이 표시된다
2. **Given** 트랙이 존재하는 상태, **When** 팬 노브를 오른쪽으로 드래그, **Then** 해당 트랙의 스테레오 위치가 우측으로 이동한다
3. **Given** 재생 중인 상태, **When** 뮤트 버튼을 클릭, **Then** 해당 트랙의 소리가 즉시 사라지고 뮤트 상태가 시각적으로 표시된다
4. **Given** 여러 트랙이 재생 중인 상태, **When** 특정 트랙의 솔로 버튼을 클릭, **Then** 해당 트랙만 들리고 나머지 트랙은 무음이 된다

---

### User Story 5 - 트랙 관리 (Priority: P5)

사용자가 트랙 헤더를 우클릭하면 컨텍스트 메뉴가 나타나고, 트랙 이름 변경, 트랙 복제, 트랙 삭제, 트랙 색상 변경 등의 작업을 할 수 있다. 트랙 헤더를 드래그하여 트랙 순서를 변경할 수 있다.

**Why this priority**: 트랙 관리는 프로젝트 정리를 위한 편의 기능으로, 핵심 워크플로우가 동작한 후에 추가해도 충분하다.

**Independent Test**: 트랙 헤더를 우클릭하여 이름을 변경하고, 드래그로 순서를 바꿀 수 있다.

**Acceptance Scenarios**:

1. **Given** 트랙이 존재하는 상태, **When** 트랙 헤더를 우클릭, **Then** 이름 변경/복제/삭제/색상 변경 메뉴가 표시된다
2. **Given** 컨텍스트 메뉴가 열린 상태, **When** "이름 변경"을 선택하고 새 이름을 입력, **Then** 트랙 헤더에 새 이름이 표시된다
3. **Given** 트랙이 2개 이상인 상태, **When** 트랙 헤더를 다른 트랙 위/아래로 드래그, **Then** 트랙 순서가 변경된다
4. **Given** 컨텍스트 메뉴에서 "삭제"를 선택, **When** 확인 다이얼로그에서 "삭제"를 클릭, **Then** 해당 트랙과 포함된 모든 리전이 제거된다

---

### Edge Cases

- 트랙이 0개인 상태에서 재생 버튼을 누르면 어떻게 되는가? → 재생 헤드만 이동, 소리 없음
- 리전을 타임라인 시작(0) 이전으로 드래그하면? → 마디 1:1에서 스냅, 더 이전으로 이동 불가
- 줌을 극단적으로 확대/축소하면? → 최소/최대 줌 레벨 제한 적용
- 100개 이상의 트랙을 추가하면? → 성능 저하 없이 스크롤로 탐색 가능해야 함 (Canvas 가상화)
- 솔로와 뮤트를 동시에 누르면? → 솔로가 뮤트보다 우선 (Logic Pro X 동작과 동일)
- 리전이 겹치도록 배치하면? → 나중에 배치된 리전이 위에 표시, 오디오는 두 리전 모두 재생

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 사용자가 MIDI 트랙을 추가/삭제할 수 있어야 한다
- **FR-002**: 타임라인은 마디/박자 기반 그리드를 Canvas 2D로 렌더링해야 한다
- **FR-003**: 재생/정지 컨트롤이 존재하며, 재생 시 재생 헤드가 BPM에 맞춰 이동해야 한다
- **FR-004**: BPM을 변경할 수 있어야 하며, 기본값은 120 BPM이다
- **FR-005**: 박자표(Time Signature)를 설정할 수 있어야 하며, 기본값은 4/4이다
- **FR-006**: 타임라인 빈 영역을 더블클릭하면 빈 리전이 생성되어야 한다
- **FR-007**: 리전을 드래그하여 시간 축/트랙 간 이동할 수 있어야 한다
- **FR-008**: 리전의 좌우 끝을 드래그하여 길이를 조절할 수 있어야 한다
- **FR-009**: 선택된 리전을 Delete 키로 삭제할 수 있어야 한다
- **FR-010**: 마우스 휠로 가로 스크롤, Cmd+휠로 줌 인/아웃이 가능해야 한다
- **FR-011**: 줌 레벨에 따라 그리드 눈금이 적절히 변해야 한다 (마디→박자→세분)
- **FR-012**: 각 트랙에 볼륨 슬라이더(-inf ~ +6dB)와 팬 노브(L100~R100)가 있어야 한다
- **FR-013**: 각 트랙에 뮤트/솔로 버튼이 있어야 한다
- **FR-014**: 재생 중 재생 헤드가 화면 밖으로 나가면 자동 스크롤해야 한다
- **FR-015**: 트랙 이름 변경, 복제, 삭제, 색상 변경이 가능해야 한다
- **FR-016**: 트랙 순서를 드래그로 변경할 수 있어야 한다
- **FR-017**: 리전 이동/크기 조절 시 마디 단위 스냅이 기본 적용되어야 한다
- **FR-018**: 위치 표시기가 현재 재생 위치를 마디:박자:틱 형식으로 표시해야 한다

### Key Entities

- **Project**: 전체 프로젝트를 나타내며 BPM, 박자표, 트랙 목록을 포함한다
- **Track**: 개별 오디오/MIDI 채널로 이름, 볼륨, 팬, 뮤트/솔로 상태, 색상, 리전 목록을 가진다
- **Region**: 트랙 위의 시간 범위를 차지하는 데이터 블록으로 시작 위치, 길이, 소속 트랙을 가진다
- **Transport**: 재생 상태(재생/정지), 현재 위치, BPM, 루프 범위 등 전역 재생 제어 정보

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 앱을 열고 30초 이내에 트랙 추가 → 리전 배치 → 재생 확인까지 완료할 수 있다
- **SC-002**: 50개 트랙 + 트랙당 20개 리전(총 1,000개 리전) 상태에서 줌/스크롤이 30fps 이상 유지된다
- **SC-003**: 재생 헤드의 시각적 위치와 실제 오디오 타이밍의 차이가 사용자가 인지할 수 없는 수준이다
- **SC-004**: 모든 주요 동작(트랙 추가, 리전 생성, 재생/정지, 볼륨/팬 조절)이 키보드 단축키로 수행 가능하다
- **SC-005**: 초보 사용자가 별도 안내 없이 트랙 추가, 리전 생성, 재생의 기본 흐름을 완료할 수 있다

## Assumptions

- Phase 1에서는 MIDI 트랙만 지원하고 오디오 트랙(녹음)은 Phase 3에서 추가한다
- Phase 1의 리전은 빈 컨테이너이며, 실제 MIDI 노트 편집은 Phase 2(피아노롤)에서 구현한다
- 재생 시 소리 출력은 Phase 2(가상 악기 연결) 이후에 가능하며, Phase 1에서는 메트로놈 클릭만 지원한다
- 프로젝트 저장/불러오기는 Phase 7에서 구현하며, Phase 1에서는 브라우저 세션 내에서만 상태 유지
- 모바일/태블릿 지원은 범위 밖이며, 데스크톱 브라우저(Chrome/Edge)만 대상으로 한다
- 인증/로그인 없이 사용 가능하다 (Phase 7에서 Supabase 연동 시 추가)
