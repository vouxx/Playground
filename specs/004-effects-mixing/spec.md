# Feature Specification: Effects & Mixing

**Feature Branch**: `004-effects-mixing`
**Created**: 2026-03-31
**Status**: Draft

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 트랙에 이펙트 추가 및 조절 (Priority: P1)

사용자가 트랙을 선택하고 이펙트 체인 패널을 열면, EQ/컴프레서/리버브/딜레이 등을 추가할 수 있다. 각 이펙트의 파라미터를 노브/슬라이더로 실시간 조절하고, 재생 중에도 즉시 소리에 반영된다. 이펙트 순서를 드래그로 변경하고, 개별 이펙트를 바이패스할 수 있다.

**Why this priority**: 이펙트는 사운드 가공의 핵심이다. EQ와 컴프레서 없이는 전문적인 믹싱이 불가능하다.

**Independent Test**: 트랙에 리버브 추가 → 파라미터 조절 → 재생하여 소리 변화 확인

**Acceptance Scenarios**:

1. **Given** 트랙이 존재하는 상태, **When** 이펙트 추가 버튼 클릭 → "리버브" 선택, **Then** 트랙 이펙트 체인에 리버브가 추가되고 기본 파라미터로 적용
2. **Given** 이펙트가 추가된 상태, **When** Decay 노브를 3초로 조절, **Then** 재생 시 리버브 꼬리가 길어짐
3. **Given** 이펙트 체인에 EQ + 리버브, **When** 순서를 리버브 → EQ로 드래그, **Then** 이펙트 처리 순서가 변경됨
4. **Given** 이펙트가 있는 상태, **When** 바이패스 버튼 클릭, **Then** 해당 이펙트가 비활성화되고 원래 소리로 복원

---

### User Story 2 - 센드/리턴 버스 (Priority: P2)

사용자가 리턴 트랙(버스)을 생성하고 리버브를 로드한다. 각 트랙에서 센드 노브로 버스에 보내는 양을 조절하면, 여러 트랙이 하나의 리버브를 공유할 수 있다.

**Why this priority**: 센드/리턴은 CPU 효율적 믹싱의 핵심이다. 트랙마다 개별 리버브 대신 공유 버스를 쓰면 리소스 절약.

**Independent Test**: 리턴 트랙에 리버브 추가 → 두 트랙에서 센드 양 조절 → 재생하여 공유 리버브 확인

**Acceptance Scenarios**:

1. **Given** DAW 상태, **When** "리턴 트랙 추가" 클릭, **Then** 리턴 타입 트랙이 생성되고 이펙트를 로드할 수 있다
2. **Given** 리턴 트랙에 리버브가 있는 상태, **When** 일반 트랙의 센드 노브를 올림, **Then** 해당 트랙 소리의 일부가 리턴 트랙 리버브를 거쳐 출력
3. **Given** 여러 트랙이 같은 리턴에 센드, **When** 재생, **Then** 모든 센드된 소리가 하나의 리버브를 공유

---

### User Story 3 - 오토메이션 커브 (Priority: P3)

사용자가 볼륨, 팬, 이펙트 파라미터에 오토메이션 커브를 그릴 수 있다. 타임라인에 오토메이션 레인이 표시되고, 포인트를 클릭/드래그로 추가/편집한다. 재생 시 커브에 따라 값이 자동으로 변한다.

**Why this priority**: 오토메이션은 동적인 믹스를 만드는 핵심이지만, 이펙트와 센드/리턴이 먼저 있어야 의미가 있다.

**Independent Test**: 볼륨 오토메이션 레인을 열고 커브를 그린 뒤 재생하여 볼륨이 변하는 것 확인

**Acceptance Scenarios**:

1. **Given** 트랙이 존재하는 상태, **When** 오토메이션 레인 표시 버튼 클릭, **Then** 트랙 아래에 오토메이션 레인이 나타나고 파라미터 선택 가능
2. **Given** 오토메이션 레인이 열린 상태, **When** 빈 영역을 클릭, **Then** 해당 시간/값 위치에 포인트가 추가됨
3. **Given** 포인트가 2개 이상, **When** 재생, **Then** 포인트 사이가 보간되어 파라미터가 자동 변화

---

### Edge Cases

- 이펙트 체인이 비어있으면? → 드라이 시그널이 직접 출력
- 센드 레벨이 0이면? → 리턴 트랙에 아무 신호도 전달되지 않음
- 오토메이션 포인트를 모두 삭제하면? → 마지막 설정값이 유지
- CPU 과부하 시? → 이펙트를 자동 바이패스하지 않고 경고 표시

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 이펙트 타입: 3밴드 EQ, 컴프레서, 리버브, 딜레이, 코러스, 디스토션
- **FR-002**: 트랙별 이펙트 체인(최대 8개) 관리 — 추가/제거/순서변경/바이패스
- **FR-003**: 각 이펙트의 파라미터를 실시간 조절할 수 있어야 한다
- **FR-004**: 리턴(버스) 트랙을 생성할 수 있어야 한다
- **FR-005**: 일반 트랙에서 리턴 트랙으로의 센드 레벨을 조절할 수 있어야 한다
- **FR-006**: 볼륨/팬/이펙트 파라미터에 오토메이션 커브를 그릴 수 있어야 한다
- **FR-007**: 재생 시 오토메이션 값이 시간에 따라 자동 적용되어야 한다
- **FR-008**: 오토메이션 포인트의 추가/삭제/이동이 가능해야 한다

### Key Entities

- **EffectInstance**: 트랙에 할당된 개별 이펙트 (타입, 파라미터 맵, 바이패스 여부)
- **EffectChain**: 트랙의 이펙트 순서 목록
- **SendConfig**: 트랙 → 리턴 트랙 센드 레벨
- **AutomationLane**: 파라미터별 오토메이션 데이터
- **AutomationPoint**: 시간(틱) + 값(0~1) 쌍

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 이펙트 파라미터 변경이 재생 중 체감 지연 없이 반영된다
- **SC-002**: 트랙당 4개 이펙트, 10트랙 동시 재생 시 오디오 끊김 없음
- **SC-003**: 오토메이션 커브가 1ms 이내 정밀도로 재생에 반영된다

## Assumptions

- Phase 1~3 기반 완성 상태
- 이펙트는 Tone.js 내장 이펙트 사용 (EQ3, Compressor, Reverb, FeedbackDelay, Chorus, Distortion)
- 오토메이션은 선형 보간(linear interpolation)을 기본으로, 추후 커브 타입 추가
- 리턴 트랙은 새로운 TrackType('return')으로 추가
