# Feature Specification: Instruments & MIDI

**Feature Branch**: `002-instruments-midi`
**Created**: 2026-03-31
**Status**: Draft
**Input**: User description: "Phase 2 악기 및 MIDI - 내장 신스, 샘플러, 드럼 머신 패드, 피아노롤 MIDI 편집, 외부 MIDI 컨트롤러 입력, 리전에 MIDI 노트 저장 및 재생"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 리전에 MIDI 노트를 입력하고 재생하기 (Priority: P1)

사용자가 타임라인에서 리전을 더블클릭하면 피아노롤 편집기가 열린다. 피아노롤에서 마우스 클릭으로 MIDI 노트를 입력하고, 재생 버튼을 누르면 내장 신스가 해당 노트를 소리로 출력한다. 노트의 위치(시간), 피치(음높이), 길이, 벨로시티를 편집할 수 있다.

**Why this priority**: MIDI 노트를 입력하고 소리를 들을 수 있어야 모든 악기/편집 기능이 의미를 가진다. DAW에서 "소리가 나는" 최초 순간이다.

**Independent Test**: 트랙 추가 → 리전 생성 → 피아노롤에서 노트 입력 → 재생하여 소리 확인

**Acceptance Scenarios**:

1. **Given** 리전이 존재하는 상태, **When** 리전을 더블클릭, **Then** 하단에 피아노롤 편집기가 열리고 해당 리전의 노트가 표시된다
2. **Given** 피아노롤이 열린 상태, **When** 빈 영역을 클릭, **Then** 해당 피치/시간 위치에 기본 길이(1/4 노트)의 MIDI 노트가 생성된다
3. **Given** 노트가 존재하는 상태, **When** 재생 버튼을 누름, **Then** 내장 신스가 노트의 피치/벨로시티에 맞춰 소리를 출력한다
4. **Given** 노트가 존재하는 상태, **When** 노트를 드래그, **Then** 피치(세로) 또는 시간(가로)이 변경된다
5. **Given** 노트가 선택된 상태, **When** 노트의 우측 끝을 드래그, **Then** 노트 길이가 변경된다
6. **Given** 노트가 선택된 상태, **When** Delete 키를 누름, **Then** 노트가 삭제된다

---

### User Story 2 - 내장 신스 선택 및 프리셋 (Priority: P2)

각 트랙에 가상 악기를 할당할 수 있다. 기본 제공 신스(사인/사각/삼각/톱니 오실레이터, FM 신스, AM 신스)와 프리셋(피아노, 패드, 베이스, 리드 등)을 선택할 수 있다. 트랙 헤더에서 악기를 변경한다.

**Why this priority**: 다양한 소리를 트랙별로 할당할 수 있어야 음악적으로 의미 있는 결과를 만들 수 있다.

**Independent Test**: 트랙 2개를 만들고 각각 다른 프리셋을 할당한 뒤 재생하여 다른 소리가 나는 것을 확인

**Acceptance Scenarios**:

1. **Given** MIDI 트랙이 존재하는 상태, **When** 트랙 헤더의 악기 선택 버튼을 클릭, **Then** 프리셋 목록(피아노, 패드, 베이스, 리드, 드럼 등)이 표시된다
2. **Given** 프리셋 목록이 열린 상태, **When** "베이스"를 선택, **Then** 해당 트랙의 신스가 베이스 사운드로 변경되고 즉시 적용된다
3. **Given** 신스가 할당된 트랙, **When** 노트가 포함된 리전을 재생, **Then** 선택된 신스 사운드로 노트가 출력된다

### User Story 3 - 드럼 머신 패드 (Priority: P3)

드럼 트랙에서는 16패드 드럼 머신 인터페이스를 사용할 수 있다. 각 패드에 드럼 샘플(킥, 스네어, 하이햇, 심벌 등)이 매핑되어 있고, 패드를 클릭하거나 키보드로 트리거할 수 있다. 패드 입력은 해당 트랙의 리전에 MIDI 노트로 기록된다.

**Why this priority**: 드럼/퍼커션은 비트메이킹의 핵심이고, 패드 인터페이스는 초보자에게 직관적인 입력 방식이다.

**Independent Test**: 드럼 트랙을 추가하고 패드를 클릭하여 소리를 확인, 녹음 모드에서 패드를 눌러 리전에 노트가 기록되는 것을 확인

**Acceptance Scenarios**:

1. **Given** 드럼 트랙이 존재하는 상태, **When** 드럼 패드 뷰로 전환, **Then** 4x4 패드 그리드가 표시되고 각 패드에 샘플 이름(Kick, Snare 등)이 표시된다
2. **Given** 패드 뷰가 열린 상태, **When** Kick 패드를 클릭, **Then** 킥 드럼 소리가 즉시 재생된다
3. **Given** 녹음 모드 + 재생 중, **When** 패드를 탭, **Then** 현재 재생 위치에 해당 드럼 노트가 리전에 기록된다
4. **Given** 드럼 리전이 존재하는 상태, **When** 재생, **Then** 기록된 드럼 패턴이 샘플 사운드로 출력된다

### User Story 4 - 외부 MIDI 컨트롤러 입력 (Priority: P4)

USB MIDI 키보드나 패드 컨트롤러를 연결하면 자동으로 감지되고, MIDI 입력이 선택된 트랙에 전달된다. 실시간 연주가 가능하고, 녹음 모드에서는 입력된 노트가 리전에 기록된다.

**Why this priority**: MIDI 컨트롤러 지원은 전문 사용자에게 핵심 기능이지만, 마우스 입력만으로도 DAW 사용이 가능하므로 후순위이다.

**Independent Test**: MIDI 키보드를 연결하고 건반을 눌러 소리가 나는 것을 확인, 녹음 모드에서 연주 후 리전에 노트가 기록되는 것을 확인

**Acceptance Scenarios**:

1. **Given** 앱이 로드된 상태, **When** USB MIDI 컨트롤러를 연결, **Then** 컨트롤러가 자동 감지되고 상태바에 연결 표시
2. **Given** MIDI 컨트롤러가 연결된 상태, **When** 건반을 누름, **Then** 선택된 트랙의 신스가 해당 피치로 소리를 출력
3. **Given** 녹음 모드 + 재생 중 + MIDI 컨트롤러 연결, **When** 건반을 연주, **Then** 연주한 노트가 현재 리전에 실시간 기록된다
4. **Given** 녹음이 완료된 상태, **When** 피아노롤을 열어 확인, **Then** 기록된 MIDI 노트가 정확한 시간/피치/벨로시티로 표시된다

---

### Edge Cases

- 피아노롤에서 노트가 리전 범위를 벗어나게 드래그하면? → 리전이 자동으로 확장
- 빈 리전(노트 0개)을 재생하면? → 소리 없이 재생 헤드만 이동
- 드럼 트랙에서 피아노롤을 열면? → 드럼 맵 피아노롤(피치 대신 악기 이름 표시)
- MIDI 컨트롤러를 재생 중에 분리하면? → 현재 소리 즉시 중단, 연결 해제 표시
- 벨로시티 0인 노트가 입력되면? → 무시 (note-off로 취급)
- 트랙에 악기가 할당되지 않은 상태에서 재생하면? → 기본 신스(사인파)로 대체

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 리전 내부에 MIDI 노트(피치, 시작 틱, 길이 틱, 벨로시티) 목록을 저장할 수 있어야 한다
- **FR-002**: 피아노롤 편집기에서 노트의 생성/삭제/이동/리사이즈가 가능해야 한다
- **FR-003**: 재생 시 Transport 타이밍에 맞춰 각 트랙의 MIDI 노트를 해당 신스로 트리거해야 한다
- **FR-004**: 트랙별로 가상 악기(신스)를 할당할 수 있어야 하며, 최소 6종 프리셋을 제공해야 한다
- **FR-005**: 신스 프리셋: 피아노, 패드, 베이스, 리드, 스트링, FM 신스
- **FR-006**: 드럼 트랙에서 16패드 드럼 머신 인터페이스를 제공해야 한다
- **FR-007**: 드럼 킷: 최소 킥, 스네어, 하이햇(오픈/클로즈), 클랩, 탐, 심벌, 퍼커션 8종
- **FR-008**: Web MIDI API를 통해 외부 MIDI 컨트롤러를 감지하고 입력을 받을 수 있어야 한다
- **FR-009**: 녹음 모드에서 MIDI 입력(마우스/패드/컨트롤러)이 리전에 실시간 기록되어야 한다
- **FR-010**: 피아노롤에서 노트 벨로시티를 조절할 수 있어야 한다
- **FR-011**: 피아노롤의 피치 축은 C0~C8 범위를 표시하고, 건반 이미지로 시각화해야 한다
- **FR-012**: 피아노롤에서 노트 이동/리사이즈 시 스냅이 적용되어야 한다

### Key Entities

- **MidiNote**: 개별 MIDI 이벤트로, 피치(0~127), 시작 틱, 길이 틱, 벨로시티(1~127)를 가진다
- **Instrument**: 트랙에 할당되는 가상 악기로, 타입(synth/sampler/drumkit)과 프리셋 이름을 가진다
- **DrumKit**: 패드 번호(0~15)와 샘플 매핑 정보를 가진다
- **MidiDevice**: 감지된 외부 MIDI 컨트롤러의 이름과 연결 상태

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 트랙 추가 → 리전 생성 → 피아노롤에서 노트 입력 → 재생으로 소리 확인까지 1분 이내에 완료할 수 있다
- **SC-002**: 64개 노트가 포함된 리전 4개를 동시 재생 시 오디오 끊김 없이 부드럽게 출력된다
- **SC-003**: 드럼 패드를 클릭했을 때 소리 출력까지의 체감 지연이 없다
- **SC-004**: MIDI 컨트롤러 연결 시 5초 이내에 자동 감지되고 즉시 사용 가능하다
- **SC-005**: 피아노롤에서 100개 이상의 노트가 있을 때 스크롤/편집이 30fps 이상 유지된다

## Assumptions

- Phase 1의 타임라인, 트랙, 리전, Transport, 믹서 기반이 완성되어 있다
- 드럼 샘플은 Tone.js 신스 기반 합성으로 제공하고, 사용자 샘플 업로드는 Phase 3에서 추가
- 신스 파라미터(ADSR, 필터 등) 세부 편집은 Phase 4(이펙트)에서 구현하며, Phase 2에서는 프리셋 선택만 지원
- Web MIDI API는 Chrome/Edge에서만 지원되며, 미지원 브라우저에서는 해당 기능이 비활성화됨
- 피아노롤은 Canvas 2D로 렌더링하며, Phase 1의 렌더링 패턴(3-레이어, rAF, 가상화)을 따른다
